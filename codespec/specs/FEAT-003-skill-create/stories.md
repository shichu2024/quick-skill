## ST-001 交互式新 Skill 创建

**作为** Skill 开发者，**我希望**通过交互式引导流程从零创建符合 Anthropic 规范的新 Skill，**以便**无需手动编写 SKILL.md 即可获得标准化的技能定义。

### 验收标准

- **AC-1**：执行创建命令后，进入交互式引导流程，依次引导用户完成以下步骤：业务分类选择 -> Skill 使命定义 -> 触发边界定义 -> 成功标准定义 -> 执行步骤定义 -> 文件生成。
- **AC-2**：业务分类步骤：展示已有的业务分类列表供用户选择，同时支持输入新的分类名称。
- **AC-3**：Skill 使命定义步骤：引导用户输入 `name`（自动校验并转换为 kebab-case 格式）和 `description`（核心能力描述）。
- **AC-4**：name 字段校验：name 不可为空，不可与已有 Skill 同名（同分类下），不符合 kebab-case 时自动转换并提示用户确认。
- **AC-5**：触发边界定义步骤：引导用户输入"When to use this"（适用场景）和"When NOT to use this"（禁止场景）。
- **AC-6**：成功标准定义步骤：引导用户输入"What to build"（产出物、规范、约束）和"Definition of done"（可量化完成标准）。
- **AC-7**：执行步骤定义步骤：引导用户输入核心执行步骤（Steps），该步骤可跳过标记为可选。
- **AC-8**：支持通过 `--category` 参数指定分类，跳过分类选择步骤直接进入使命定义。
- **AC-9**：交互流程中每一步提供清晰的提示信息和输入示例，降低用户理解成本。

### 范围外

- 不支持批量创建
- 不支持非交互式参数化创建（仅交互式）

## ST-002 交互式已有 Skill 编辑

**作为** Skill 开发者，**我希望**通过 `--edit` 参数对已有 Skill 进行交互式编辑，**以便**在不直接修改文件的情况下迭代优化 Skill 定义。

### 验收标准

- **AC-1**：通过 `--edit [skill-name]` 指定要编辑的 Skill，工具自动在 `./skills/` 所有分类子目录中查找匹配的 Skill。
- **AC-2**：加载已有 SKILL.md 内容，按章节展示当前值，用户可逐章节修改或保留不变。
- **AC-3**：编辑流程展示的章节顺序与创建流程一致，每个章节预填当前值。
- **AC-4**：用户可跳过任意章节（保留原值），至少修改一个章节才触发文件更新。
- **AC-5**：编辑模式下支持修改 Skill 所属的业务分类。
- **AC-6**：写入更新前自动备份上一版本 SKILL.md 到同目录下。
- **AC-7**：指定的 skill-name 不存在时，输出明确的错误提示并终止执行。

### 范围外

- 不支持同时编辑多个 Skill
- 不在编辑流程中修改 evals/ 目录下的测试用例（由 eval-sync 联动处理）

## ST-003 SKILL.md 标准文件生成与更新

**作为** Skill 开发者，**我希望**创建或编辑完成后自动生成/更新符合 Anthropic 官方规范的 SKILL.md 文件，**以便**无需关心文件格式和章节结构即可获得标准化输出。

### 验收标准

- **AC-1**：创建模式下，自动在 `./skills/[业务分类]/[skill-name]/` 目录下生成 SKILL.md 文件。
- **AC-2**：生成的 SKILL.md 严格遵循标准格式：YAML front matter（name、description）+ When to use this + When NOT to use this + What to build + Steps + Definition of done。
- **AC-3**：YAML front matter 中的 name 为 kebab-case 格式，description 为用户输入的核心能力描述。
- **AC-4**：创建模式下，同步创建配套的 `evals/` 空目录。
- **AC-5**：编辑模式下，仅更新用户修改的章节，未修改章节保持原内容不变。
- **AC-6**：生成的 SKILL.md 文件编码为 UTF-8，换行符与项目配置一致。
- **AC-7**：文件生成/更新完成后，输出确认信息：文件路径、生成/更新状态。

### 范围外

- 不生成测试用例文件（由 eval-sync 联动处理）
- 不自动部署到 Agent 目录

## ST-004 编辑后 eval-sync 联动触发

**作为** Skill 开发者，**我希望**在创建或编辑 Skill 完成后自动提示是否同步更新测试用例，**以便**技能定义的变更能即时反映到测试覆盖中。

### 验收标准

- **AC-1**：创建新 Skill 完成后，提示"是否立即生成测试用例"，用户确认后自动调用 eval-gen 模块。
- **AC-2**：编辑已有 Skill 完成后，提示"是否同步更新测试用例"，用户确认后自动调用 eval-sync 模块。
- **AC-3**：用户拒绝或跳过联动提示时，不执行任何用例操作，不阻塞 Skill 创建/编辑流程的完成。
- **AC-4**：联动调用 eval-sync / eval-gen 失败时，输出错误提示但不影响 Skill 创建/编辑的最终结果。
- **AC-5**：联动提示仅在 SKILL.md 实际发生变化（新建或内容修改）时出现，未做修改时不触发。

### 范围外

- 不在联动中执行评测运行
- 不强制用户必须同步用例

## ST-005 创建中断恢复与草稿保存

**作为** Skill 开发者，**我希望**在交互式创建流程中意外中断后能从上次的位置继续，**以便**已输入的内容不会丢失。

### 验收标准

- **AC-1**：交互式创建流程中，用户每完成一个步骤，自动保存当前进度为草稿。
- **AC-2**：草稿保存的内容包括：已完成的分类选择、name、description、各已填写章节的内容。
- **AC-3**：用户重新执行创建命令时，检测到存在未完成的草稿，提示用户选择"继续上次"或"重新开始"。
- **AC-4**：选择"继续上次"后，从草稿中断的步骤继续，已完成的步骤预填草稿内容。
- **AC-5**：选择"重新开始"后，清除草稿，进入全新创建流程。
- **AC-6**：Skill 创建成功完成后，自动清除对应的草稿数据。

### 范围外

- 不支持编辑模式的中断恢复
- 草稿不跨会话持久化（仅当前 CLI 会话内有效）
