## ST-001 单技能全维度诊断扫描

**作为** Skill 维护者，**我希望**对指定路径的存量技能执行 7 个维度的合规性扫描，**以便**快速了解该技能在结构、元数据、边界、标准、格式、评测、兼容性方面存在的全部问题。

### 验收标准

- **AC-1**：给定一个有效的存量技能路径，诊断工具对该路径下的所有文件执行 7 个维度的扫描，输出每个维度的检查结果（通过 / 不通过 / 不适用）。
- **AC-2**：结构合规性维度检查：技能必须为独立目录，包含核心技能文件，无零散散落文件。
- **AC-3**：元数据合规性维度检查：技能必须有明确的 name 和 description 定义。
- **AC-4**：边界合规性维度检查：技能必须定义"When to use this"和"When NOT to use this"。
- **AC-5**：标准合规性维度检查：技能必须有可量化的"Definition of done"。
- **AC-6**：格式合规性维度检查：技能文件命名符合 kebab-case 规范，无特殊字符和空格。
- **AC-7**：评测合规性维度检查：技能必须具备覆盖 4 类核心场景的测试用例。
- **AC-8**：兼容性合规性维度检查：技能无单一 Agent 强绑定逻辑，可跨 Agent 通用。
- **AC-9**：每个不通过维度标注修复等级（必须修复 / 建议修复），并给出具体的不通过原因描述。
- **AC-10**：当指定路径不存在或无法访问时，输出明确的错误提示并终止执行。

### 范围外

- 不执行任何修复操作
- 不评估技能核心业务逻辑的正确性

## ST-002 量化合规评分输出

**作为** Skill 维护者，**我希望**诊断完成后获得一个量化的合规评分（0-100 分），**以便**直观判断技能的整体合规水平，并与团队其他成员对齐改造优先级。

### 验收标准

- **AC-1**：诊断完成后输出一个 0-100 范围内的合规总分。
- **AC-2**：每个诊断维度分别输出独立评分，维度评分加权汇总为总分。
- **AC-3**："必须修复"维度的权重高于"建议修复"维度，评分算法对必须修复项的不通过施加更高扣分。
- **AC-4**：所有维度均通过时，总分为 100 分。
- **AC-5**：评分结果在 CLI 终端中清晰展示，包含总分和各维度分数。

### 范围外

- 不支持用户自定义评分权重（V1 采用内置默认权重）
- 不生成分数趋势图

## ST-003 诊断报告可视化输出

**作为** Skill 维护者，**我希望**诊断结果以结构化的可视化报告形式输出，**以便**将报告分享给团队成员或存档留档。

### 验收标准

- **AC-1**：支持 Markdown 格式的诊断报告输出，包含合规评分、各维度诊断结果、风险项高亮、改造优先级排序。
- **AC-2**：报告按"必须修复 -> 建议修复 -> 已通过"优先级排序展示各诊断维度。
- **AC-3**：报告包含诊断的时间戳、技能路径、技能名称等元信息。
- **AC-4**：支持通过 `--output` 参数自定义报告输出路径。
- **AC-5**：未指定 `--output` 时，报告默认输出到当前工作目录。

### 范围外

- V1 不支持 HTML 格式报告
- 不生成 PDF 格式

## ST-004 精准改造清单生成

**作为** Skill 维护者，**我希望**诊断完成后获得一份精准的改造清单，明确区分"可自动修复"和"需人工确认"两类问题，**以便**快速规划改造工作并判断哪些可一键修复。

### 验收标准

- **AC-1**：诊断完成后输出改造清单，清单中每项包含：问题描述、涉及维度、修复等级、是否可自动修复、预期效果。
- **AC-2**：改造清单按"可自动修复 / 需人工确认"两类分组展示。
- **AC-3**：可自动修复项明确列出修复动作（如重命名文件、补充缺失目录），需人工确认项给出处理建议。
- **AC-4**：改造清单集成在诊断报告中作为独立章节输出。
- **AC-5**：当所有诊断项均通过时，改造清单为空，输出"无需改造"提示。

### 范围外

- 不执行实际修复操作（由 ST-006 负责）
- 不生成修复脚本文件

## ST-005 批量诊断与汇总报告

**作为** Skill 仓库维护者，**我希望**对指定目录下所有存量技能执行批量诊断，输出汇总报告，**以便**快速评估整体合规状况并排序改造优先级。

### 验收标准

- **AC-1**：给定一个目录路径，工具扫描该目录下所有子目录，对每个子目录执行技能诊断。
- **AC-2**：批量诊断完成后输出汇总报告，包含：总扫描数、合规数、不合规数、按合规评分从低到高排序的技能清单。
- **AC-3**：汇总报告中每个技能包含：技能名称、合规评分、不通过维度列表。
- **AC-4**：支持通过 `--filter` 参数按技能文件类型筛选需要诊断的技能。
- **AC-5**：批量诊断过程中单个技能诊断失败不阻塞其他技能的诊断，失败项标记错误原因。
- **AC-6**：批量诊断过程展示实时进度反馈（已扫描数 / 总数）。

### 范围外

- 不支持并发数配置（V1 采用串行执行）
- 不支持暂停与续跑

## ST-006 可标准化问题自动修复

**作为** Skill 维护者，**我希望**通过 `--fix-auto` 参数一键自动修复诊断发现的可标准化问题，**以便**无需人工干预即可快速解决格式和命名类合规问题。

### 验收标准

- **AC-1**：使用 `--fix-auto` 参数后，工具在诊断完成后自动执行所有标记为"可自动修复"的修复动作。
- **AC-2**：自动修复前，自动备份原始文件到 `.backup/` 目录，备份内容可完整还原。
- **AC-3**：自动修复仅限于可标准化的项（如文件重命名、目录结构补齐），不修改技能核心业务内容。
- **AC-4**：自动修复完成后，输出修复清单：已修复项数量、修复内容摘要、备份路径。
- **AC-5**：自动修复失败时，回滚已执行的修复操作，输出失败原因，不影响原始文件状态。
- **AC-6**：标记为"需人工确认"的项不执行自动修复，在修复清单中明确标注"需人工处理"。

### 范围外

- 不修复需要人工判断的内容（如业务逻辑描述、触发边界定义）
- 不生成独立的修复脚本文件
